# Charge order in the half-filled bond-Holstein model

Code and data for the manuscript

> C. Jordan, G. Issa, E. Khatami, R. Scalettar, B. Cohen-Stead, S. Johnston,
> *Charge order in the half-filled bond-Holstein model*, Phys. Rev. B (accepted).

The repository contains:

- the Julia DQMC simulation script (`julia/bond_holstein_square.jl`) used to
  generate the data, built on top of
  [SmoQyDQMC.jl](https://github.com/SmoQySuite/SmoQyDQMC.jl),
- the resulting electron-density snapshots used as input to the
  machine-learning analyses (PCA, t-SNE, learning by confusion, and the
  CNN method of Broecker, Assaad and Trebst),
- the LBC test-accuracy outputs, and
- self-contained Python scripts to reproduce panels of Fig. 6 and Fig. 7
  of the paper.

## Coupling parameters

The dimensionless bond-Holstein coupling is

$$\lambda_{\rm bond} = \frac{4\,\alpha^{2}}{\omega_{0}^{2}\,W},\qquad W = 8t,$$

so for $\omega_{0} = 1$ (the value used throughout the data set) one has

$$\alpha = \sqrt{2\,\lambda},\qquad 1/\lambda = 2/\alpha^{2}.$$

| $\alpha$ (file naming) | $1/\lambda$ |
|-----------------------:|------------:|
| 0.6325                 | 5           |
| 0.7071                 | 4           |
| 0.8165                 | 3           |
| 1.0000                 | 2           |
| 1.4142                 | 1           |

## Setup

The pipeline uses two separate environments: a Python environment for the
machine-learning analysis and figure scripts, and a Julia environment for the
DQMC simulation.  Each is independent — you only need the one corresponding
to the work you're doing.

### Python environment (PCA / t-SNE / LBC / figures)

Create the `bond-holstein` conda environment from `environment.yml` and
activate it:

```bash
mamba env create -f environment.yml      # or: conda env create -f environment.yml
mamba activate bond-holstein              # or: conda activate bond-holstein
```

If conda/mamba is not available, the same dependencies can be installed into
any Python ≥ 3.11 environment with

```bash
pip install -r requirements.txt
```

A note on Apple Silicon: importing TensorFlow and PyTorch in the same
interpreter session segfaults.  Each script in this repo uses only one of
them, so this is not an issue when running them individually.

### Julia environment (DQMC simulation)

Install Julia ≥ 1.10 (we recommend [`juliaup`](https://github.com/JuliaLang/juliaup))
and instantiate the project pinned in `julia/Project.toml`:

```bash
julia --project=julia -e 'using Pkg; Pkg.instantiate()'
```

This downloads SmoQyDQMC.jl together with MPI.jl and JLD2.jl at the versions
recorded in `julia/Manifest.toml`.  Subsequently every Julia command must be
run with `--project=julia` so the right environment is picked up; equivalently,
from inside `julia/` you can use `--project=.` or just enter the package
manager with `]activate .`.

To run an MPI-parallel simulation you also need `mpiexecjl`:

```bash
julia --project=julia -e 'using MPI; MPI.install_mpiexecjl(force=true)'
```

This places `mpiexecjl` under `~/.julia/bin/`; make sure that directory is on
your `PATH`.

## Running the DQMC simulation

`julia/bond_holstein_square.jl` runs a determinant quantum Monte Carlo
simulation of the half-filled bond-Holstein model on an `L x L` square lattice,
using SmoQyDQMC.jl with combined HMC + reflection + global-swap updates for
the phonon fields.  Local real-space observables (electron density and bond
phonon displacements) are written as CSV files under
`<datafolder>/local_density/position/...` and
`<datafolder>/phonon_position/position/...`, one file per measurement bin.
The standard SmoQyDQMC measurement container (Green's function, structure
factors, pair / spin / charge correlations) is also written.

CLI arguments (positional, in this order):

| arg | meaning                                              |
| --- | ---------------------------------------------------- |
|  1  | `sID` – simulation ID                                |
|  2  | `Ω` – phonon frequency                               |
|  3  | `α` – e-ph coupling                                  |
|  4  | `β` – inverse temperature                            |
|  5  | `L` – linear system size                             |
|  6  | `N_burnin` – thermalization sweeps                   |
|  7  | `N_updates` – measurement sweeps (must be `% N_bins == 0`) |
|  8  | `N_bins` – number of measurement bins                |
|  9  | `cdw_start` – `0` or `1`; whether to seed phonons in a CDW pattern |

Single-process run:

```bash
julia --project=julia julia/bond_holstein_square.jl 0 1.0 1.0 4.0 12 5000 10000 50 0
```

MPI-parallel run with 8 walkers:

```bash
mpiexecjl --project=julia -n 8 julia julia/bond_holstein_square.jl \
    0 1.0 1.0 4.0 12 5000 10000 50 0
```

The combined electron-density CSVs that the Python figure scripts consume
(one row per snapshot, last column the sweep parameter) are produced
downstream from the per-bin local-density files; once a sweep over `β` (or
over `1/λ`) has been collected, concatenate the corresponding bins to obtain
the format described under `data/`.

## Reproducing the figures

All commands below assume the repo root as the working directory and the
Python `bond-holstein` environment is active.

### Fig. 6 (a)-(c): PCA and t-SNE at fixed coupling

```bash
python scripts/figure6_pca_tsne.py \
    --w 1.0 --alpha 1.0 --L 12 --sID 7 \
    --perplexity 40 --seed 0
```

Outputs:

- `figures/figure6a_pca_scatter.pdf`
- `figures/figure6b_pca_average.pdf`
- `figures/figure6c_tsne_embedding.pdf`

The notebook `PCA+tsne.ipynb` reproduces the same figures with intermediate
visualizations of representative snapshots and the PCA eigenvalue spectrum.

### Fig. 6 (d): LBC W-shape

The plotting script accepts one or more accs CSV files and an explicit sweep
grid:

```bash
python scripts/plot_lbc_accuracy.py \
    data/accuracies/accs_w1.00_a1.0000_L12_seed*-4.csv \
    --grid 0.15 1.10 --xlabel beta \
    --out figures/lbc_accuracy_a1.0_sID4.pdf
```

Use `--xlabel '1/lambda'` if the sweep is over the inverse coupling instead.

### Fig. 7: phase diagram

```bash
python scripts/figure7_phase_diagram.py
```

Reads the reference (method, 1/lambda, T) values from
`data/published_phase_diagram.csv` and writes
`figures/figure7_phase_diagram.pdf`.

## Re-running the machine-learning training

To reproduce the LBC accs files from raw electron-density snapshots, e.g.
for `alpha = 1.0`, `sID = 4`:

```bash
python scripts/lbc_train.py \
    data/electron_densities_w1.00_a1.0000_L12_csv-4/electron_densities_w1.00_a1.0000_L12-4.csv \
    runs/lbc_w1.00_a1.0000_seed2000-4 \
    --tc-min 0.15 --tc-max 1.10 --num-tc 20 \
    --w 1.0 --alpha 1.0 --L 12 --sID 4 --seed 2000 \
    --num-epochs 150
```

For Fig. 6(d) (sweep over `1/lambda` at fixed `beta`):

```bash
python scripts/lbc_train.py \
    data/electron_densities_w1.00_b0.30_L12_csv-0/electron_densities_w1.00_b0.30_L12-0.csv \
    runs/lbc_b0.30 \
    --tc-min 1.0 --tc-max 2.5 --num-tc 16 \
    --w 1.0 --L 12 --sID 0 --seed 2000 \
    --num-epochs 150
```

For the Broecker/Assaad/Trebst CNN sweep (one or more values of
`nskip` per coupling):

```bash
python scripts/broecker_train.py \
    data/electron_densities_w1.00_b0.30_L12_csv-0/electron_densities_w1.00_b0.30_L12-0.csv \
    runs/broecker_b0.30 \
    --L 12 --sID 1 --nskip 6
```

## Quick reference: activating each environment

```bash
# Python (PCA, t-SNE, LBC, plotting)
mamba activate bond-holstein
python scripts/figure6_pca_tsne.py
python scripts/figure7_phase_diagram.py
python scripts/plot_lbc_accuracy.py data/accuracies/accs_*.csv --grid 0.15 1.10 --xlabel beta
python scripts/lbc_train.py <input.csv> <out_dir> --tc-min ... --tc-max ... --num-tc ...
python scripts/broecker_train.py <input.csv> <out_dir> --L 12 --sID 1 --nskip 6
mamba deactivate

# Julia (DQMC simulation)
julia --project=julia julia/bond_holstein_square.jl <args...>
mpiexecjl --project=julia -n <N> julia julia/bond_holstein_square.jl <args...>
```
